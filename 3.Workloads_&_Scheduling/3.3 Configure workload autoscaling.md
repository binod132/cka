## Configure workload autoscaling.

#### Install metrics server
```sh
wget https://github.com/kubernetes-sigs/metrics-server/releases/latest/download/components.yaml
```
##### Edit components.yaml 
```
- --kubelet-insecure-tls # on container args and kubectl apply -f components.yaml
```
```sh
kubectl apply -f components.yaml
kubectl top pods
```

#### Demo: Autoscaling a Deployment
##### Create deployment with resource defined
```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  labels:
    app: nginx-hpa
  name: nginx-hpa
spec:
  replicas: 1
  selector:
    matchLabels:
      app: nginx-hpa
  strategy: {}
  template:
    metadata:
      labels:
        app: nginx-hpa
    spec:
      containers:
      - image: nginx
        name: nginx-hpa
        resources: 
          requests:
            memory: "64Mi"
            cpu: "50m"
          limits:
            memory: "128Mi"
            cpu: "100m"
status: {}
```
```sh
kubectl apply -f deployment.yaml    
```
```sh
kubectl expose deploy nginx-hpa --port=80
```
```sh
kubectl autoscale deployment nginx-hpa --cpu-percent=50 --min=1 --max=3
```

or

```sh
kubectl autoscale deployment nginx-hpa --cpu=50% --memory=70% --min=1 --max=2 --dry-run -o yaml
```
```sh
kubectl run -it --rm load-test --image=busybox -- /bin/sh -c "while true; do wget -q -O- http://nginx-hpa; done"
```

kubectl get hpa -w
